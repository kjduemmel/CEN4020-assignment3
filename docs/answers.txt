Inheritance is a class can reuse and extend another class by inheriting its fields/methods
Polymorphism is one function/method being defined again to behave differently with a different object type.
Encapsulation is bundling data and methods together in a class and controlling access to the internal state with "private" etc..

9:
In Dictator and Lieutenants, there’s a single “dictator” (project lead) who makes the final merge decisions, and lieutenants each manage a subsystem, collect changes from contributors, and forward curated sets of commits up to the dictator.

In an Integration Manager workflow, the key role is a separate integration manager who pulls and merges contributions (often from many developers’ public branches/forks) into the mainline. It’s less about a hierarchy of subsystem lieutenants and more about one person/team responsible for integrating everyone’s work.

10:
With 100 developers, Dictator and Lieutenants scales better than Centralized because:

Work is merged first within subsystems by lieutenants, so the mainline isn’t constantly hit by 100 simultaneous changes.

Lieutenants review and curate changes for their area, so code review and decision-making isn’t a single bottleneck.

Each subsystem has a responsible maintainer who understands that code deeply and can keep it consistent.

Only vetted, tested sets of changes move upward, reducing “everyone pushes to main” breakages.

The dictator handles big-picture integration decisions, while lieutenants handle the details—so the lead isn’t overwhelmed by every small change, having decisions be faster to make.
